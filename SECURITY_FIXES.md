# 安全修复说明

修复日期：2026-07-15

## 修复内容

- 对静态资源路径进行安全解码和目录边界校验，避免畸形 URL 导致服务进程退出或发生路径穿越。
- 只信任受控反向代理追加的客户端地址，防止伪造 `X-Forwarded-For` 绕过接口限流。
- 创建、加入和退出车队接口统一返回脱敏数据，不再暴露成员 `openid`。
- 开发代理仅监听本机，并限制协议和目标来源，防止代理目标被外部请求覆盖。
- 微信群验证有效期由 180 天缩短至 30 天。
- 清理部署和排障文档中的服务器地址、账号、凭证路径及其他运行环境信息。
- 将 Taro 统一升级至 4.2.0、Webpack 升级至安全兼容版本，并约束存在安全公告的传递依赖版本。
- 将微信开发者工具的 WXML 压缩配置纳入项目配置。

## 验证结果

- 畸形资源路径返回 HTTP 400，服务仍可继续响应健康检查。
- 使用变化的伪造来源地址无法绕过登录接口限流，超过限制后返回 HTTP 429。
- 开发代理拒绝绝对形式的外部目标地址。
- `pnpm audit --prod` 检查结果为 0 个生产依赖漏洞。
- H5 与微信小程序构建均成功，最终 `dist` 为微信小程序产物。

## 后续注意事项

- 历史提交中可能仍保留旧部署信息；如需彻底从远程仓库移除，需要单独执行历史改写并协调所有协作者重新同步。
- 开发工具链仍包含部分上游旧依赖安全公告，它们不进入生产小程序产物。不要使用当前工具链处理来源不可信的代码或压缩包，并在上游发布兼容版本后继续升级。
